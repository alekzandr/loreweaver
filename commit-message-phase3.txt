## GitHub Actions Workflow

Created `.github/workflows/content-validation.yml`:
- **Triggers:** PRs to main with changes in content-submissions/, schemas/, or validation scripts
- **Jobs:**
  - `validate-content`: Validates submissions and posts PR comments
  - `test-validation-suite`: Runs full validation test suite

### Validate Content Job Steps:
1. JSON linting with GitHub Actions summary
2. Schema validation on changed files
3. Production cross-check for duplicate detection
4. Quality report with improvement suggestions
5. PR comment with detailed results (creates or updates existing)
6. Merge blocking on validation failures

## Content Merge Script

Created `scripts/merge-content.js`:
- Merges validated submissions into production data files
- Auto-detects content type from structure
- Generates IDs automatically if missing
- Checks for duplicates before merge
- Maintains alphabetical ordering by ID
- Updates metadata (lastUpdated, totalCount)

### Merge Script Options:
- `--dry-run`: Preview changes without writing files
- `--all`: Merge all submissions in content-submissions/
- `--file PATH`: Merge specific file only

### Merge Process:
1. Loads submission and detects type
2. Loads production data
3. Checks for duplicates
4. Generates ID if needed
5. Adds to production array
6. Sorts alphabetically
7. Updates metadata
8. Writes to production JSON
